# JavaScript

  • We write modern JavaScript that we eventually transpile to support all the browsers we have to (see Browsers Support).
  • We manage dependencies with npm (opens new window).
  • We use EsLint (opens new window) and Prettier (opens new window) to ensure consistency.

# Code styling

We combine a set of EsLint rules (to avoid errors, ensure security and best practices) with Prettier (to enforce a certain style).

# Setup

Install dependencies:

npm i -D eslint prettier eslint-config-prettier eslint-plugin-prettier @liip-lausanne/eslint-config

NOTICE

We prefer JavaScript configuration files over JSON so we can add comments and logic if necessary.

Create an EsLint config file named .eslintrc.js at the project root with:

module.exports = {
  env: {
    browser: true,
    es2021: true,
  },
  extends: ['@liip-lausanne', 'plugin:prettier/recommended'],
  parserOptions: {
    ecmaVersion: 2021,
    sourceType: 'module',
  },
};

Create a Prettier config file named prettier.config.js at the project root with:

module.exports = {
  singleQuote: true,
  trailingComma: 'es5',
};

Add some scripts in your package.json to validate/format files:

// ...
"scripts": {
  // ...
  "eslint": "eslint .",
  "eslint:fix": "npm run eslint -- --fix",
  // ...
}
// ...

See scripts for conventions.

# Testing

We prefer Jest (opens new window) to run unit & functional tests and Cypress (opens new window) for end-to-end tests.

We enjoy msw (opens new window) to mock asynchronous requests.

# Third-parties

Our go-to framework is Vue.js (opens new window) with its friends Vuex (opens new window) and Vue Router (opens new window) if necessary. In the case of a SPA, we use Vue CLI (opens new window) to scaffold the project.

Libraries we often use accross projects:

  • Lodash (opens new window) for utility methods
  • Axios (opens new window) for asynchronous requests
  • date-fns (opens new window) to work with time and dates
  • accounting.js (opens new window) to work with money and currencies
